IITC プラグインディレクトリ

Niantic Ingress Intel マップ向けの IITC カスタムプラグインと拡張ツールをまとめた一覧です。

このディレクトリでは、ポータル追跡、フィールド計画、ドローン経路計画、活動ログ分析、リチャージ監視など、Ingress エージェント向けの IITC ユーティリティを紹介しています。

GitHub リポジトリを見る

プラグイン一覧

各ページでは、プラグインの用途、どんなエージェントに向いているか、そして Ingress Intel マップ上の計画や分析をどう支援するかを説明しています。

Portal DB

IITC 用の永続的なローカルポータルデータベースを構築し、座標移動検知や更新統計を提供します。

Portal Afterimage

公式マーカーがズームで隠れたときに、既知ポータルの記憶レイヤーを描画します。

プレイヤー活動ログ

COMM からエージェントの行動を記録し、移動軌跡をマップ上で可視化して振り返りを助けます。

リチャージモニター

ポータルの健康状態をリアルタイムで監視し、防衛計画向けに減衰タイミングを予測します。

Uniques Tools

訪問、取得、ドローン移動の記録を強化し、公式履歴の取り込みにも対応します。

Homogeneous Fields

同質コントロールフィールドの計画と可視化を支援し、構造的なフィールド運用に役立ちます。

Drone Flight Planner

S2 メカニクス、キー制約、JSON 入出力を使ってドローン経路を計画します。

Fanfield Planner

交差しない底辺リンクやアンカー反射を考慮しながら、2 フェーズのファンフィールドを計画します。

All Portal Names

指定ズームでポータル名を強制表示し、重なりがあっても Intel マップ上のラベルを見やすくします。

Polygon AP Counter

ユーザーが描いた多角形内の Portal 数、Link、Field、ENL または RES の潜在 AP を分析します。

これらの IITC プラグインを使う理由

これらのツールは、Ingress Intel マップをより深く活用したいエージェント向けに作られており、ポータル記録、作戦計画、フィールド構築、COMM の振り返り、日常運用の効率化を支援します。

  • ポータル追跡とローカル保存でマップ認識を強化できます。
  • ドローン移動、ファンフィールド、同質フィールドの計画精度を高められます。
  • Intel マップ上の情報を、より実用的な行動判断につなげられます。

よくある質問

  • IITC とは何ですか？ IITC は Ingress Intel マップを拡張するインターフェースで、視認性、操作性、計画機能を高めたいエージェントに適しています。
  • これらは Niantic の Ingress 向けプラグインですか？ はい。このディレクトリは Niantic Ingress Intel マップ向けの IITC プラグインを扱っており、Kubernetes やネットワーク分野の Ingress とは無関係です。
  • ソースコードやスクリプトはどこで確認できますか？ 上部の GitHub リポジトリリンクから、ソースコード、プラグインスクリプト、更新履歴を確認できます。
注: これらのプラグインはローカルで受信したデータのみに基づき、Niantic の利用規約に違反しません。
