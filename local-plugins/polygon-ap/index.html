Polygon AP Counter

An area analysis plugin for IITC on Niantic's Ingress Intel Map.

Polygon AP Counter analyzes portals, links, and fields inside closed polygons drawn on the Intel Map. It helps Ingress agents estimate potential AP for both ENL and RES, review portal counts, and quickly compare multiple target areas.

Who This Plugin Is For

This plugin is useful for agents who want to evaluate target areas before farming, clearing, or rebuilding. It is especially helpful when you want quick AP estimates for both factions across multiple user-drawn polygons without manually counting portals, links, and fields.

Key Features

  • Polygon-based portal analysis: Detects all portals inside polygons created with Draw Tools.
  • Faction AP calculation: Estimates potential AP separately for Enlightened and Resistance.
  • Map bubble labels: Shows compact polygon stats directly on the map.
  • Summary table: Lists each polygon with its own counts and totals for quick comparison.
  • Refresh and view controls: Recompute stats after edits and center the map on a chosen polygon.

How to Use Polygon AP Counter

  1. Make sure Draw Tools is installed and enabled.
  2. Draw one or more closed polygons on the map.
  3. Open Polygon AP Counter from the IITC toolbox.
  4. Review the dialog showing stats for each polygon.
  5. Use Refresh if you move the map or change the polygon shapes.
  6. Use View to center the map on a selected polygon.

AP Calculation Logic

  • Capture and deploy: Includes AP for capturing neutral portals and fully deploying 8 resonators.
  • Destruction: Includes AP for destroying enemy resonators, links, and fields.
  • Machina reclaim: Includes reclaim AP for Machina-controlled portals.
  • Links and fields: Counts links and fields when at least one vertex is inside the polygon boundary.

The calculation model is intentionally close to standard Ingress AP rules, similar in spirit to Available AP Statistics.

Requirements and Limits

This plugin requires Draw Tools. Without any drawn polygons, there is nothing to analyze and the dialog will not show useful results.

Because it works from the portal, link, and field data currently available to your IITC client, the output should be treated as a practical planning estimate rather than a guaranteed live global truth.

FAQ

  • Does this plugin work without Draw Tools? No. It depends on user-drawn polygons created through Draw Tools.
  • Why are ENL and RES AP shown separately? The available AP depends on the faction ownership of portals, links, and fields inside the selected area.
  • Can the numbers change after I move the map? Yes. If the visible data changes or you edit polygons, use Refresh to recompute the results.
