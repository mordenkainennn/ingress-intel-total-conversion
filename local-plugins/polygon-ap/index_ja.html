Polygon AP Counter

Niantic Ingress Intel Map 向けのエリア分析プラグインです。

Polygon AP Counter は、Draw Tools で描いた閉じた多角形内の Portal、Link、Field を分析するためのプラグインです。Ingress エージェントが、ENL と RES の両陣営について、そのエリアで見込める AP をすばやく比較できるようにします。

GitHub でこのプラグインのディレクトリを見る | すべての IITC プラグインを見る

クレジット： 元の Polygon Portal Counter は 57Cell によって作られ、AP 統計機能は Cloverjune により追加・保守されています。

このプラグインが向いている人

ファーム、クリア、再構築の前に対象エリアを評価したいエージェント向けです。複数の候補地について、Portal 数や Link / Field、両陣営の AP 期待値を手動で数えずに比較したいときに便利です。

主な機能

  • 多角形ベースの Portal 分析： Draw Tools で描いたポリゴン内の Portal を自動抽出します。
  • 陣営別 AP 計算： Enlightened と Resistance の潜在 AP を個別に計算します。
  • 地図上ラベル： 各ポリゴンの簡易統計をマップ上に表示します。
  • サマリーテーブル： ポリゴンごとの統計と総計を一覧で比較できます。
  • Refresh / View 操作： 多角形編集後の再計算や、対象ポリゴンへのジャンプが可能です。

使い方

  1. Draw Tools が有効であることを確認します。
  2. 地図上に 1 つ以上の閉じた多角形を描きます。
  3. IITC toolbox から Polygon AP Counter を開きます。
  4. 各ポリゴンの統計をダイアログで確認します。
  5. 地図やポリゴンを変更したら Refresh で再計算します。
  6. View を押すと、その多角形が地図中央に来ます。

AP 計算ロジック

  • キャプチャとデプロイ： 中立 Portal の確保と 8 本フルデプロイ の AP を含みます。
  • 破壊： 敵 Resonator、Link、Field の破壊 AP を含みます。
  • Machina 回収： Machina 支配 Portal の回収 AP も含まれます。
  • Link / Field 判定： 頂点の少なくとも 1 つが多角形内にある Link / Field をカウントします。

計算の考え方は、標準的な Ingress AP ルールにできるだけ沿っており、Available AP Statistics に近い発想です。

必要条件と制限

このプラグインには Draw Tools が必要です。多角形が描かれていなければ分析対象がないため、有効な結果は表示されません。

また、IITC クライアントが現在保持している Portal、Link、Field データに基づく推定なので、結果は実運用向けの目安として扱うのが適切です。

FAQ

  • Draw Tools がなくても使えますか？ いいえ。ユーザーが描いた多角形に依存しています。
  • なぜ ENL と RES を分けて表示するのですか？ エリア内の Portal、Link、Field の所有陣営によって得られる AP が変わるためです。
  • 地図を動かしたら数字は変わりますか？ はい。可視データや多角形が変わったら、Refresh で再計算してください。

関連プラグイン
