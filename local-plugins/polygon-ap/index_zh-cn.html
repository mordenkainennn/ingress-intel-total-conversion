多边形 AP 统计

适用于 Niantic Ingress Intel Map 的区域分析插件。

Polygon AP Counter 用来分析 Draw Tools 绘制的封闭多边形内的 Portal、Link 和 Field。它可以帮助 Ingress 特工快速估算 ENL 与 RES 两个阵营在该区域内可能获得的 AP，并对多个目标区域进行横向比较。

适合哪些用户

这个插件适合在行动前评估目标区域的玩家，比如刷 AP、清场、重建区域，或者比较多个地块的收益潜力。它特别适合那些不想手动数 Portal、Link 和 Field，而希望快速得到双方阵营 AP 预估值的人。

核心功能

  • 基于多边形的 Portal 分析： 自动识别 Draw Tools 多边形内的所有 Portal。
  • 双阵营 AP 计算： 分别估算 启蒙军 和 抵抗军 的潜在 AP。
  • 地图气泡标签： 在地图上直接显示每个多边形的紧凑统计信息。
  • 摘要表格： 按多边形列出详细统计，并提供总览。
  • 刷新与定位： 支持在修改多边形后重新计算，并快速跳转到目标区域。

如何使用

  1. 确认已经安装并启用 Draw Tools。
  2. 在地图上绘制一个或多个封闭多边形。
  3. 从 IITC toolbox 打开 Polygon AP Counter。
  4. 查看对话框中每个多边形的统计结果。
  5. 如果移动地图或修改了多边形，点击 Refresh 重新计算。
  6. 点击 View 将地图中心跳到对应多边形。

AP 计算逻辑

  • 占领与部署： 计入占领中立 Portal 以及完整部署 8 个 Resonator 的 AP。
  • 摧毁： 计入摧毁敌方 Resonator、Link 和 Field 的 AP。
  • Machina 回收： 计入回收 Machina 控制 Portal 的 AP。
  • Link 与 Field 规则： 只要 Link 或 Field 至少有一个顶点落在多边形边界内，就会被纳入统计。

这套计算思路尽量贴近标准的 Ingress AP 规则，与 Available AP Statistics 的思路相近。

依赖与限制

这个插件依赖 Draw Tools。如果没有绘制多边形，就没有可分析的区域，因此对话框也不会显示有意义的数据。

由于它基于当前 IITC 客户端已经获取到的 Portal、Link 和 Field 数据进行估算，所以结果更适合作为行动规划参考，而不是绝对实时、全局完整的真值。

常见问题

  • 没有 Draw Tools 也能用吗？ 不能。它依赖 Draw Tools 提供的用户绘制多边形。
  • 为什么 ENL 和 RES 要分开显示？ 因为区域内 Portal、Link 和 Field 的阵营归属不同，双方可以获得的 AP 潜力也不同。
  • 移动地图后数字会变化吗？ 会。如果可见数据变化了，或者你修改了多边形，需要重新点击 Refresh 计算。
