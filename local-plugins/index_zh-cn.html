IITC 插件目录

面向 Niantic Ingress Intel 地图的 IITC 自定义与增强插件集合。

这个目录汇集了适合 Ingress 特工的地图增强工具，包括 Portal 追踪、场规划、无人机路线规划、活动分析、充电监控，以及其它围绕 Intel 地图的 IITC 实用插件。

查看 GitHub 仓库

插件目录

下方每个页面都会说明插件的用途、适合的使用场景，以及它如何帮助你在 Ingress Intel 地图上进行规划、分析或日常操作。

Portal DB

为 IITC 建立持久化的本地 Portal 数据库，支持坐标变动检测和更新统计，适合 Ingress Intel 数据分析。

Portal Afterimage

当官方标记因缩放而隐藏时，为已见过的 Portal 绘制记忆残影图层。

玩家活动日志

记录 COMM 中的特工活动，并在地图上可视化展示移动轨迹，便于回顾 Ingress 行动。

充电监控器

实时监控 Portal 健康度，并为防守规划预测衰减时间窗口。

Uniques Tools

增强访问、占领和无人机移动记录，支持官方历史导入，适合长期追踪 Ingress 进度。

Homogeneous Fields

帮助规划与可视化同质控制场，适用于结构化的 Ingress 场作业。

Drone Flight Planner

基于 S2 机制规划无人机路线，并支持钥匙跳转限制与 JSON 导入导出。

扇形场规划器

规划双阶段扇形场，处理无交叉底边、锚点反射，并输出分阶段总结。

All Portal Names

在指定缩放等级强制显示 Portal 名称，让 Intel 地图标签即使重叠也尽量可见。

多边形 AP 统计

分析用户绘制多边形内的 Portal 数量、Link、Field 和启蒙军或抵抗军的潜在 AP。

为什么使用这些 IITC 插件

这些工具面向希望扩展 Ingress Intel 地图能力的特工，重点覆盖 Portal 记忆、行动规划、场构建、COMM 回顾和日常操作效率提升。

  • 通过 Portal 追踪和本地数据存储提升地图感知能力。
  • 为无人机跳跃、扇形场和同质场提供更强的规划支持。
  • 把 Intel 地图上的活动记录转换为更易操作的情报信息。

常见问题

  • 什么是 IITC？ IITC 是 Ingress Intel 地图的增强界面，适合希望获得更多控制能力、可视化信息和规划工具的特工。
  • 这些插件是给 Niantic 的 Ingress 用的吗？ 是的。这个目录专门介绍适用于 Niantic Ingress Intel 地图的 IITC 插件，而不是 Kubernetes 或网络软件相关的 Ingress。
  • 在哪里查看源码和插件文件？ 你可以通过上方的 GitHub 仓库链接查看源代码、插件脚本和更新记录。
注：这些插件仅基于本地接收的数据，不与服务器额外交互，不违反 Niantic 的服务条款。
