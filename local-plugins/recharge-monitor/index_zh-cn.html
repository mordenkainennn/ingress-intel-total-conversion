Recharge Monitor

面向 Niantic Ingress Intel 地图的 IITC Portal 监控列表、健康追踪与衰减预测插件。

Recharge Monitor 帮助特工跨区域追踪重要 Portal 的健康度，在 Portal 不在当前视野时估算衰减时间，并通过本地 COMM 与活动历史恢复部署时间。

在 GitHub 上查看此插件目录 | 查看全部 IITC 插件

适合哪些用户

如果你需要维护多个重要 Portal、长链锚点或跨区域防守目标，这个插件会非常实用。尤其当这些 Portal 并不总在你的当前地图视野中时，它能提供持续的本地监控能力。

核心功能

  • 监控列表面板：可从 Portal 详情面板将目标加入 watchlist，并集中管理。
  • Portal 健康度追踪：查看当前能量、最后观测状态，以及部署时间相关信息。
  • 衰减预测：根据标准 15% 日衰减模式，估算视野外 Portal 何时会中立化。
  • 历史恢复：可从 COMM 回溯和 Player Activity Log 中恢复更准确的部署时间。
  • 充电提醒：当 watchlist 中的 Portal 健康度跌破阈值时，在工具箱显示预警数量。

历史同步与时间恢复

Recharge Monitor 是否准确，很大程度取决于它能否还原 Portal 的原始部署时间。为此它支持两种本地历史来源：

  • COMM 回溯：扫描 IITC 本地聊天缓存中的 capture 与 deploy 事件。
  • Player Activity Log 集成：通过 GUID 和坐标匹配读取兼容的活动历史。

因为插件依赖本地历史，所以“预计衰减”是否准确，取决于你的客户端过去是否实际看到了这些相关事件。

如何使用

  1. 选择一个 Portal，并通过详情面板把它加入 watchlist。
  2. 从工具箱中的 Recharge Mon 打开主面板。
  3. 当你需要补全历史时，使用 Sync History 从本地来源恢复部署时间。
  4. 当工具箱变成橙色或红色时，就表示某些监控 Portal 已进入危险区间。

高手提示：Recharge Monitor 与 Player Activity Log 搭配使用效果最佳，因为它可以在你加入 Portal 时自动吸收本地活动历史中的部署时间信息。

重要提示：历史恢复与衰减预测都依赖本地数据来源。如果你的客户端当时没有看到相关事件，估算结果就可能不完整。

提醒：这是一个本地规划与监控工具，不是服务器端的绝对历史真相来源，不应被当作权威数据库。

常见问题

  • 它能预测当前视野外 Portal 的衰减吗？ 可以，只要插件拥有足够的本地历史来推断部署时间与当前状态。
  • 为什么推荐搭配 Player Activity Log？ 因为它能提供额外的本地 capture / deploy 历史，从而提升 Recharge Monitor 的估算质量。
  • 为什么预测结果仍可能不准？ 因为它依赖你本地看到并记录下来的 COMM 与历史数据，缺失事件就意味着缺失上下文。
