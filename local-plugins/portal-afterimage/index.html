Portal Afterimage

Prerequisite: This plugin requires the Portal DB plugin to be installed and enabled. Afterimage relies on Portal DB for all portal coordinates and timestamps.

Portal Afterimage draws a subtle memory layer of portals you have already seen, but only when the official portal layer is hidden by zoom. It never guesses or fills missing data — it only visualizes what your client has actually received.

This plugin depends on Portal DB for portal coordinates and last-seen timestamps. Afterimage itself stores only GUID and name locally.

Features

  • Memory Layer Only

    Displays faint afterimages only when official portals are hidden by zoom. It never replaces or modifies native portal rendering.

  • Local & Private

    All data stays in your browser. Nothing is uploaded or shared across devices.

  • S2 Cell Abstraction

    Portals are grouped by S2 cells (configurable level). Each cell shows a limited number of portals to avoid overload.

  • Safe Degradation

    Hard cap (MAX_DRAWN_ELEMENTS) prevents heavy areas from freezing the map.

Settings

  • S2 Level (15–18): Higher level = smaller cells = more detail.
  • Per Cell (1–3): How many portals to display inside each S2 cell.
  • Selection Rule: Portals are chosen by lastSeen from Portal DB (most recent first).

S2 Level Reference (L15–L19)

Approximate sizes shown below are for latitude ~37°. Actual sizes vary by latitude.

Level E-W Edge N-S Edge Approx Area Typical Use
L15 ~509 m ~637 m ~0.32 km² Map data loading boundary
L16 ~254 m ~318 m ~0.08 km² Control Field rendering precision
L17 ~127 m ~159 m ~0.02 km² Pokémon GO POI density limit
L18 ~63 m ~79 m ~0.005 km² Fine geofencing
L19 ~32 m ~40 m ~1,280 m² Portal placement uniqueness check

Tip: Start at L18 with Per Cell = 1, then increase level or per-cell count based on density. Higher detail means more markers and faster hitting the 5000 cap.
