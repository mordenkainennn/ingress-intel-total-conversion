扇形场规划器

适用于 Niantic Ingress Intel Map 的 anchor-based 扇形场规划插件。

Fanfield Planner 用于帮助 Ingress 特工规划 anchor-based fanfield 和 pincushion 风格方案。它支持先扫描三角区域内的 Portal，再筛选底边点集合，并生成双阶段执行路线、统计摘要、图层覆盖和文字步骤，同时还支持可选的 SBUL 外连限制。

适合哪些用户

如果你需要做结构化的 anchor fanfield，而不是临场随手拉线，这个插件会很适合你。它尤其适合希望把准备阶段和反射阶段分开处理、需要精确控制底边 Portal 集合、并且关心外连预算的玩家。

当前重点： Phase 1 会基于筛选后的候选点构建一个不交叉的底边图。Phase 2 则回到 Anchor 向这张底边图反射。你可以选择忽略外连上限，也可以启用 SBUL 限制模式，让插件在给定上限内尽量获得更多 field。

核心功能

  • 三角区域规划： 选择 1 个 Anchor 和 2 个外框点，定义规划三角形。
  • Portal 筛选： 扫描三角形内当前已加载 Portal，并通过 Included / Excluded 列表筛选底边集合。
  • 双阶段执行： 先生成不交叉的 Phase 1 底边图，再生成从 Anchor 出发的 Phase 2 反射计划。
  • SBUL 限制： 可选启用 0 到 4 个 SBUL 的 Phase 2 限制，并在外连预算内尽量最大化 field 数。
  • 地图图层覆盖： 分别绘制两个阶段的 links、fields 和高亮图层。
  • 可读文字输出： 输出易执行的步骤说明、路线摘要和钥匙需求。

使用流程

  1. 在 IITC toolbox 中打开 Fanfield Planner。
  2. 选择 Anchor，再选择两个外三角顶点。
  3. 点击 Scan Triangle 扫描三角形内已加载的 Portal。
  4. 在 Included 和 Excluded 列表之间移动 Portal，整理底边集合。
  5. 如果需要，启用 Limit Phase 2 by SBUL 并选择 0 到 4。
  6. 点击 Plan Fanfield 生成图层、摘要和钥匙需求。

算法概览

Phase 1

底边 Portal 会围绕 Anchor 按角度排序，并沿较短方向访问。每访问一个底边点，就先连回 Anchor，再在不产生交叉的前提下尽量回连更多历史底边点。最终生成的图，就是 Phase 2 真实反射所依据的结构。

Phase 2

规划器回到 Anchor 后，会向 Phase 1 的底边图反射。在不受限模式下，会尽量发出更多 Anchor link；在 SBUL 限制模式下，最大外连数为 8 + 8 * SBUL，插件会在这个预算内选择能闭合最多 field 的反射集合与顺序。

常见问题

  • Phase 1 和 Phase 2 的区别是什么？ Phase 1 负责构建不交叉的底边图，Phase 2 则回到 Anchor，对这个底边图进行反射来闭合场。
  • 为什么需要 Included / Excluded 列表？ 它能让你精细控制参与底边构建的 Portal 集合，而不需要重画三角形或手动逐个选点。
  • SBUL 限制有什么用？ 它限制 Phase 2 可用的外连数量，规划器会在这个预算内尽量获得更多 field。
