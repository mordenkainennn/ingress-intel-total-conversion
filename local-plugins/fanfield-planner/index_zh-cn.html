扇形场规划器

在 GitHub 上查看此插件目录

Fanfield Planner 是一个用于 IITC 的扇形场与 pincushion 规划插件。它支持先定义三角形范围，再扫描候选 Portal、整理 Included / Excluded 列表，最后生成双阶段执行路线、图层覆盖与文字计划。

当前重点： Phase 1 会基于筛选后的候选点构建一个不交叉的底边图。Phase 2 则回到 Anchor 向这张底边图反射。你可以选择忽略外连上限，也可以启用 SBUL 限制模式，让插件在给定上限内尽量获得更多 field。

主要功能

  • 选择 1 个 Anchor 和 2 个外框点，定义用于规划的外三角。
  • 扫描三角形内当前已加载的所有 Portal，并通过 Included / Excluded 列表筛选候选底边点。
  • 支持锁定三角形顶点，避免在审查候选点时误改 Anchor 或外框点。
  • 支持在列表中选中 Portal 后直接在地图上高亮定位。
  • 生成不交叉的 Phase 1 底边构建路线。
  • 基于真实的 Phase 1 底边图生成 Phase 2 反射计划。
  • 可选启用 Phase 2 的 SBUL 限制模式，支持 0 到 4 个 SBUL，并在可用外连预算内尽量最大化 field 数量。
  • 分别绘制两个阶段的 links / fields 图层，并输出可执行的文字步骤。

使用流程

  1. 在 toolbox 中打开 Fanfield Planner。
  2. 选择 Anchor，并再选择两个外三角顶点。
  3. 点击 Scan Triangle 扫描三角形内当前已加载的 Portal。
  4. 在 Included 和 Excluded 两个列表之间移动 Portal，整理出想要的底边集合。
  5. 如有需要，启用 Limit Phase 2 by SBUL 并选择 0 到 4 的 SBUL 数量。
  6. 点击 Plan Fanfield 生成路线、图层、统计摘要和钥匙需求。

算法概览

Phase 1

底边点会先围绕 Anchor 按角度排序，再选择总路程较短的方向进行访问。每访问一个底边点，规划器都会先连回 Anchor，再在不产生交叉的前提下尽量回连更多历史底边点。Phase 1 形成的这张图，就是 Phase 2 真正反射所依据的底边图。

Phase 2

规划器回到 Anchor 后，会向 Phase 1 的底边图反射。在不受限模式下，会尽可能反射更多 Anchor link。在 SBUL 限制模式下，Phase 2 的最大外连数为 8 + 8 * SBUL，插件会在这个上限内贪心选择能闭合最多 field 的反射子集和顺序。
