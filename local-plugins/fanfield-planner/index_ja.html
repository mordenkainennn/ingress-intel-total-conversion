Fanfield Planner

GitHub でこのプラグインのディレクトリを見る

Fanfield Planner は、IITC 上でアンカー型ファンフィールドや pincushion 構成を計画するためのプラグインです。三角形を定義し、その内部の候補 Portal を走査して整理し、2 フェーズの実行計画、マップオーバーレイ、テキスト手順を生成できます。

現在の重点: Phase 1 では、選別した候補から交差しないベースグラフを構築します。Phase 2 ではアンカーへ戻ってそのグラフに反射します。外向きリンク数の制限を無視することも、SBUL 制限モードを有効にして上限内で field 数を最大化することもできます。

主な機能

  • アンカー 1 点と外枠 2 点を選び、計画対象の三角形を定義できます。
  • 三角形内で現在読み込まれているすべての Portal を走査し、Included / Excluded リストで候補を整理できます。
  • 候補確認中に Anchor や外枠点を誤って変更しないよう、三角形頂点をロックできます。
  • リスト内で選んだ Portal をマップ上でハイライト表示できます。
  • 交差しない Phase 1 のベース構築ルートを生成します。
  • 実際の Phase 1 ベースグラフに基づく Phase 2 の反射計画を生成します。
  • Phase 2 の SBUL 制限モードを任意で有効化でき、SBUL 0 から 4 の範囲で外向きリンク予算内の field 数最大化を試みます。
  • 各フェーズの links / fields を専用レイヤーに描画し、実行しやすいテキスト手順も出力します。

使用手順

  1. toolbox から Fanfield Planner を開きます。
  2. Anchor を選択し、続けて外三角の頂点 2 点を選びます。
  3. Scan Triangle を押して、三角形内の現在読み込み済み Portal を収集します。
  4. Included と Excluded の間で Portal を移動し、意図したベース集合に調整します。
  5. 必要なら Limit Phase 2 by SBUL を有効化し、0 から 4 の値を選びます。
  6. Plan Fanfield を押して、ルート、オーバーレイ、集計、必要キー数を生成します。

アルゴリズム概要

Phase 1

ベース Portal はアンカー周りの角度で並べ替えられ、移動距離が短い向きで訪問されます。各ベース点はまずアンカーへリンクし、その後、交差しない範囲で過去のベース点へできるだけ多くバックリンクします。ここで作られたグラフが、そのまま Phase 2 の反射対象になります。

Phase 2

アンカーへ戻った後、Phase 1 で構築したベースグラフに向けて反射します。無制限モードでは可能な限り多くのアンカーリンクを投げます。SBUL 制限モードでは、最大外向きリンク数を 8 + 8 * SBUL に制限し、その枠内でより多くの field を閉じられる反射対象と順序を貪欲に選択します。
