Fanfield Planner

Fanfield Planner is an IITC plugin for building anchor based fanfields and pincushion style plans. It lets you select one anchor portal and a set of base portals, then generates a two phase execution route with map overlays and readable text output.

Current focus: Phase 1 builds a non-crossing base graph. Phase 2 returns to the anchor and reflects links into that graph to close as many fields as possible. The summary also reports per-phase counts, total AP, and SBUL requirements for heavy Phase 2 reflection.

What It Does

  • Select one anchor portal and multiple base portals directly from the IITC map.
  • Build a Phase 1 route that creates a non-crossing base structure.
  • Run a Phase 2 reflection plan from the anchor against the actual Phase 1 base graph.
  • Draw links and fields on dedicated IITC layers for both phases.
  • Generate step-by-step text with distances, bearings, key hints, per-phase counts, total AP, and SBUL reminders.

Planning Workflow

  1. Open Fanfield Planner from toolbox.
  2. Select Anchor mode and click the top portal.
  3. Select Base mode and click the base portals.
  4. Press Plan Fanfield to generate the two phase route.
  5. Inspect the text output and the four map layers, then iterate if needed.

Algorithm Overview

Phase 1

Base portals are sorted around the anchor and traversed in the shorter direction. Each visited base links back to the anchor and greedily backlinks to historical bases while rejecting crossing links.

Phase 2

The planner returns to the anchor and chooses a reflection order that closes as many supported fields as possible against the Phase 1 base graph. If outgoing links exceed the normal 8-link limit, the plan warns about required SBUL mods.

Map Layers

  • Fanfield Plan (Links Phase 1)
  • Fanfield Plan (Fields Phase 1)
  • Fanfield Plan (Links Phase 2)
  • Fanfield Plan (Fields Phase 2)
