Player Activity Log

This plugin monitors public COMM messages and stores parsed player activities in your browser.

Use it to review actions over time, visualize movement paths, and exchange merged data between users.

What's New in 0.8.3

  • Added toolbox Portal Held Time display: Activity Log（xxD） for selected owned portals.
  • Improved update behavior when switching portals, and hid held-time label for neutral portals.
  • Added disclaimer text in the Activity Log window.

Features

  • Automatic Logging from COMM

    Passively listens to public COMM events and records supported actions (capture, deploy, link, field, etc.). MACHINA is ignored.

  • Interactive Activity Viewer

    Open from toolbox and browse players on the left with detailed activity entries on the right. Search supports player ID/name filtering.

  • Portal Held Time in Toolbox

    When you select an owned portal, the toolbox link shows held time as Activity Log（xxD）. If no local capture record is found, or the portal is neutral, it stays Activity Log.

  • Import and Export

    Import supports merge with de-duplication. Export JSON and CSV are available.

  • Local Storage

    All data is stored locally in browser localStorage.

Important: Data is local only and not synced across devices/accounts. Back up regularly using export features.

Disclaimer: The data source is local recorded history and may be incomplete or inaccurate.

Compatibility Note: Trail drawing overlaps with the official Player activity tracker layer. Disable one of them if needed.
