Portal DB

Persistent Infrastructure Plugin for IITC

About Portal DB

Portal DB is a foundation plugin designed to solve the "Out of View" problem in IITC. It provides a local, persistent database for portal coordinates and basic status using IndexedDB.

Persistent Assets

Data is stored permanently in your browser. It doesn't disappear when you reload or move away, creating a long-term data asset for your Ingress activities.

Fact Layer

Treats coordinates as "Strong Facts" and ownership as "Best Effort", providing a reliable source of truth for planning and analysis.

API Documentation

Other plugins should use the window.plugin.portalDB API to access data. All methods are asynchronous and return a Promise.

getPortal(guid)

Fetch a portal record by its GUID.

const portal = await window.plugin.portalDB.getPortal('abc.123');
if (portal) {
    console.log(`Team: ${portal.team}, Location: ${portal.latE6}, ${portal.lngE6}`);
}

getPortalsInBounds(bounds)

Get all cached portals within a specific geographic area (Leaflet LatLngBounds).

const bounds = window.map.getBounds();
const portals = await window.plugin.portalDB.getPortalsInBounds(bounds);
console.log(`Found ${portals.length} portals in database for this area.`);

getStats()

Get database statistics.

const stats = await window.plugin.portalDB.getStats();
console.log(`Total portals stored: ${stats.count}`);

关于 Portal DB

Portal DB 是一个旨在解决 IITC “视野外数据缺失”问题的基础设施插件。它利用 IndexedDB 为 Portal 坐标和基础状态提供本地持久化数据库。

持久化资产

数据永久存储在浏览器中。不会随页面刷新或视野移动而消失，为您的 Ingress 活动建立长期的本地数据资产。

事实层

将坐标视为“强事实”，将归属关系视为“尽力而为”，为规划和分析提供可靠的事实源。

API 调用指南

其他插件应通过 window.plugin.portalDB API 访问数据。所有方法均为异步，并返回 Promise。

getPortal(guid)

通过 GUID 获取 Portal 记录。

const portal = await window.plugin.portalDB.getPortal('abc.123');
if (portal) {
    console.log(`阵营: ${portal.team}, 坐标: ${portal.latE6}, ${portal.lngE6}`);
}

getPortalsInBounds(bounds)

获取特定地理区域（Leaflet LatLngBounds）内所有已缓存的 Portal。

const bounds = window.map.getBounds();
const portals = await window.plugin.portalDB.getPortalsInBounds(bounds);
console.log(`该区域数据库中存有 ${portals.length} 个 Portal。`);

getStats()

获取数据库统计信息。

const stats = await window.plugin.portalDB.getStats();
console.log(`数据库共存储了 ${stats.count} 个 Portal。`);

關於 Portal DB

Portal DB 是一個旨在解決 IITC 「視野外數據缺失」問題的基礎設施插件。它利用 IndexedDB 為 Portal 座標和基礎狀態提供本地持久化數據庫。

持久化資產

數據永久存儲在瀏覽器中。不會隨頁面刷新或視野移動而消失，為您的 Ingress 活動建立長期的本地數據資產。

事實層

將座標視為「強事實」，將歸屬關係視為「盡力而為」，為規劃和分析提供可靠的事實源。

API 調用指南

其他插件應通過 window.plugin.portalDB API 訪問數據。所有方法均為異步，並返回 Promise。

getPortal(guid)

通過 GUID 獲取 Portal 記錄。

const portal = await window.plugin.portalDB.getPortal('abc.123');
if (portal) {
    console.log(`陣營: ${portal.team}, 座標: ${portal.latE6}, ${portal.lngE6}`);
}

getPortalsInBounds(bounds)

獲取特定地理區域（Leaflet LatLngBounds）內所有已快取的 Portal。

const bounds = window.map.getBounds();
const portals = await window.plugin.portalDB.getPortalsInBounds(bounds);
console.log(`該區域數據庫中存有 ${portals.length} 個 Portal。`);

getStats()

獲取數據庫統計信息。

const stats = await window.plugin.portalDB.getStats();
console.log(`數據庫共存儲了 ${stats.count} 個 Portal。`);
