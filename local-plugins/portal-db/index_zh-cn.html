Portal DB

Portal DB 是 IITC 生态系统的一个基础架构插件。它的设计目的是在您的浏览器中构建一个本地化、持久化的 Portal 基础信息数据库。

通过在您浏览地图时自动收集数据，它能够为其他需要屏幕外 Portal 坐标和阵营信息的插件提供可靠的“单一事实来源 (Single Source of Truth)”。

功能特性

  • 持久化本地存储

    使用 IndexedDB 永久存储 Portal 数据（GUID、经纬度、阵营、最后可见时间）。这些数据在不同会话和浏览器重启后依然保留。

  • 被动数据采集

    在您浏览地图时自动捕获 Portal 信息。它直接拦截原始游戏实体数据，确保高性能和广泛的数据覆盖，无需任何手动操作。

  • Portal 移动检测 🛡️

    自动检测 Portal 的坐标变动。如果某个 Portal 的移动距离超过 3 米，插件会记录其原始位置、新位置以及发现时间。这对于防御通过申请移动 Portal 来破坏 Block Link 的战术非常有效。

  • 视觉预警提醒

    当检测到新的 Portal 移动时，侧边栏工具箱中的 "Portal DB" 链接会变为橙红色并显示警告数量（例如：⚠️ Portal Moved (3)）。

  • 更新统计 (Debug)

    内置详细的小时桶统计系统，记录数据的更新原因（新增、变更、刷新、跳过）。支持实时查看过滤效率。

API 参考

此插件通过全局对象 window.plugin.portalDB 暴露其功能。所有数据访问方法均为异步并返回 Promise。

getPortal(guid)

通过 GUID 获取单个 Portal 记录。

// 示例：检查 Portal 是否存在并打印其坐标
window.plugin.portalDB.getPortal('your-portal-guid')
  .then(portal => {
    if (portal) {
      console.log(`找到: ${portal.latE6}, ${portal.lngE6}`);
    } else {
      console.log('本地数据库中未找到该 Portal');
    }
  });

getPortalsInBounds(bounds)

获取指定地理范围内的所有 Portal。用于查找特定区域（例如当前地图视图）内的所有 Portal。

// 示例：查找当前地图视图内的所有已存储 Portal
const bounds = map.getBounds(); // Leaflet LatLngBounds 对象
window.plugin.portalDB.getPortalsInBounds(bounds)
  .then(results => {
    console.log(`在此区域找到 ${results.length} 个 Portal。`);
    results.forEach(p => console.log(p.guid, p.team));
  });

返回值对象结构

所有方法均返回符合以下标准格式的 Portal 对象：

{
  "guid": "...",           // 字符串: Portal 的 GUID
  "latE6": 35689500,       // 数字: 纬度 * 1,000,000
  "lngE6": 139691700,      // 数字: 经度 * 1,000,000
  "team": "E",             // 字符串: 'E' (启), 'R' (蓝), 'N' (白), 'M' (红)
  "lastSeen": 1707000000   // 数字: 最后更新的时间戳 (毫秒)
}

如何使用

  1. 只需浏览 Intel 地图！插件会在后台静默运行，收集 Portal 数据并检测其移动。
  2. 如果侧边栏链接变红，说明有 Portal 被移动了。点击 "Portal DB" 查看详情。
  3. 在弹出的对话框中，"Moved Portals" 列表会展示所有移动记录。点击列表项可直接在地图上跳转到该 Portal。
  4. 使用 Export (导出) 和 Import (导入) 按钮来备份或迁移您的数据。

开发者须知: 此插件主要作为后端服务使用。如果您正在开发一个插件，需要知道某个当前未渲染 GUID 的“位置在哪里？”，Portal DB 就是解决方案。
