Portal DB

Portal DB is a foundational infrastructure plugin for the IITC ecosystem. It is designed to create a local, persistent database of Portal basic information within your browser.

By automatically collecting data as you browse the map, it serves as a reliable "Single Source of Truth" for other plugins that require portal coordinates and team information, even when those portals are not currently visible on the screen.

Features

  • Persistent Local Storage

    Uses IndexedDB to store Portal data (GUID, Latitude, Longitude, Team, Last Seen Time) permanently. This data persists across sessions and browser restarts.

  • Passive Data Collection

    Automatically captures portal information from the map as you browse. It intercepts raw game entities to ensure high performance and extensive coverage without requiring manual action.

  • Data Management

    Includes a simple management interface in the toolbox to Export (backup) and Import (restore/share) your JSON database.

API Reference

This plugin exposes its functionality via the global object window.plugin.portalDB. All data access methods are asynchronous and return a Promise.

getPortal(guid)

Retrieves a single portal record by its GUID.

// Example: Check if a portal exists and log its coordinates
window.plugin.portalDB.getPortal('your-portal-guid')
  .then(portal => {
    if (portal) {
      console.log(`Found: ${portal.latE6}, ${portal.lngE6}`);
    } else {
      console.log('Portal not found in local DB');
    }
  });

getPortalsInBounds(bounds)

Retrieves all portals within a given geographical boundary. Useful for finding portals in a specific area (e.g., current map view).

// Example: Find all stored portals in the current map view
const bounds = map.getBounds(); // Leaflet LatLngBounds
window.plugin.portalDB.getPortalsInBounds(bounds)
  .then(results => {
    console.log(`Found ${results.length} portals in this area.`);
    results.forEach(p => console.log(p.guid, p.team));
  });

Return Object Structure

Methods return portal objects in the following standardized format:

{
  "guid": "...",           // String: The Portal's GUID
  "latE6": 35689500,       // Number: Latitude * 1,000,000
  "lngE6": 139691700,      // Number: Longitude * 1,000,000
  "team": "E",             // String: 'E' (Enl), 'R' (Res), 'N' (Neutral), 'M' (Machina)
  "lastSeen": 1707000000   // Number: Timestamp (ms) of last update
}

How to Use

  1. Just browse the Intel Map! The plugin runs silently in the background, collecting data for every portal loaded into your view.
  2. To manage your data, click the "Portal DB" link in the IITC toolbox sidebar.
  3. A dialog will appear showing the total count of portals in your local database.
  4. Use the Export and Import buttons to backup or transfer your data.

Note for Developers: This plugin is primarily intended as a backend service. If you are developing a plugin that needs to know "where is this portal?" for a GUID that isn't currently rendered, Portal DB is the solution.
