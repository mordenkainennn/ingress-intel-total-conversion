Portal Coverage Layers

Niantic Ingress Intel Map 向けの IITC カバレッジフィルタープラグインです。

Portal Coverage Layers はローカルのポータル履歴を 2 つの実用的なレイヤーに変換します。Unvisited Portals と Uncaptured Portals を使うことで、ヒートマップを読み解く代わりに、すでに完了したポータルを隠し、次に行くべき対象だけを地図上に残せます。

GitHub でこのプラグインのディレクトリを見る | すべての IITC プラグインを見る

依存プラグイン: このプラグインは Uniques Tools、つまり local-plugins/uniques-tools に依存しています。Uniques Tools が保存する visited / captured 履歴がない場合、Portal Coverage Layers は表示対象を判定できません。

このプラグインが向いている人

密度を見るヒートマップよりも、次に処理すべきポータルを直接見たいエージェントに向いています。uniques の埋め残し確認、未キャプチャポータルの回収、作戦後の後片付けルート作成などで特に読みやすい表示になります。

主な機能

  • 2 つの専用レイヤー: Unvisited Portals と Uncaptured Portals を別々の IITC レイヤーとして追加します。
  • 履歴ベースのフィルタリング: Uniques Tools が管理している visited と captured の状態をそのまま利用します。
  • 判断が速い: 完了済みポータルを隠すため、地図に残るポータルが次の目標になります。
  • 両レイヤー同時利用: 両方を有効化した場合、どちらかの条件に合うポータルは表示されたままになります。
  • 自動更新: マップデータ変更時や Uniques Tools の状態更新時に自動で再適用されます。

各レイヤーが表示するもの

  • Unvisited Portals: visited でも captured でもないポータルだけを表示します。
  • Uncaptured Portals: captured ではないポータルを表示します。つまり、訪問済みでも未キャプチャなら残ります。
  • 両方を有効化した場合: どちらかの条件を満たすポータルは表示され続けるため、未キャプチャ対象を広く残しつつ、完全未訪問の候補も同時に把握できます。

使い方

  1. まず Uniques Tools を導入して有効化します。
  2. Uniques Tools にポータル履歴を登録します。手動入力でもインポートでも構いません。
  3. Portal Coverage Layers を導入して有効化します。
  4. IITC のレイヤー一覧から Unvisited Portals、Uncaptured Portals、またはその両方を有効にします。
  5. 地図上に残ったポータルを、そのままルート計画や回収対象として使います。

ヒートマップより直感的な理由

ヒートマップは全体の傾向や密度を見るのに向いていますが、色の強弱や重なりを自分で解釈する必要があります。Portal Coverage Layers はもっと直接的で、完了済みポータルを地図から消すことで、表示されているもの自体を行動リストに変えます。

そのため、最後の uniques 埋め、過去エリアの再確認、まだ未訪問や未キャプチャが残っているかのチェックに特に有効です。

要件と制限

このプラグインには Uniques Tools が必要です。公式履歴を自分で取得したり、独自データベースを持ったりはせず、local-plugins/uniques-tools に保存された履歴だけを解釈して表示します。

また、現在 IITC クライアントに読み込まれているポータルと、ローカルに保存されている履歴に基づいて動作するため、全世界の完全な最新状態を示すものではなく、実用的な行動用ビューとして使うのが適しています。

FAQ

  • Uniques Tools なしでも動きますか？ いいえ。visited / captured 履歴は Uniques Tools に依存しています。
  • Unvisited と Uncaptured の違いは何ですか？ Unvisited は visited と captured の両方を隠し、Uncaptured は captured だけを隠します。
  • 2 つのプラグインで別々にデータを管理する必要がありますか？ ありません。履歴更新は Uniques Tools 側だけで行い、このプラグインはそれを読み取るだけです。

関連プラグイン

  • Uniques Tools は必須の依存プラグインであり、visited / captured 履歴の元データです。
  • Portal Afterimage は既知ポータルの視認性向上に役立ち、このプラグインと組み合わせやすいです。
  • Player Activity Log はローカル行動履歴を補完し、ポータル回収の計画に役立ちます。
