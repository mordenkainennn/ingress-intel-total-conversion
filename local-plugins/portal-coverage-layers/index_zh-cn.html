Portal Coverage Layers

适用于 Niantic Ingress Intel Map 的 IITC 目标筛选图层插件。

Portal Coverage Layers 会把你的本地 Portal 历史转换成两个非常直接的地图图层：Unvisited Portals 与 Uncaptured Portals。相比需要自己理解冷热分布的热力图，这个插件会直接隐藏已经完成的 Portal，让地图上剩下的点就是你下一步该去处理的目标。

在 GitHub 上查看此插件目录 | 查看全部 IITC 插件

依赖说明： 这个插件优先使用 Uniques Tools，也就是 local-plugins/uniques-tools，但也支持回退到 IITC 原版 plugins/uniques.js。如果两者都没有安装，或者都没有保存 visited / captured 历史数据，那么 Portal Coverage Layers 就没有可用的筛选依据。

适合哪些用户

如果你更想要一个“直接告诉你下一步该去哪”的地图，而不是一个需要自己判断密度的趋势图，那么这个插件会很适合你。它特别适合补 uniques、清理未占领 Portal、复盘旧行动区域，或者规划一条专门补漏的路线。

核心功能

  • 两个独立图层： 为 IITC 增加 Unvisited Portals 和 Uncaptured Portals 两个独立 layer。
  • 基于历史数据筛选： 直接读取 Uniques Tools 中维护的 visited 和 captured 状态；若未安装，则回退读取原版 Uniques 插件数据。
  • 决策更直接： 把已经完成的 Portal 隐藏后，地图上剩下的就是待处理目标。
  • 支持双图层并开： 两个 layer 同时启用时，只要 Portal 满足任一条件就会保留显示。
  • 自动刷新： 地图数据变化，或任一支持的 uniques 插件更新状态后，会自动重新应用筛选。

两个图层分别显示什么

  • Unvisited Portals： 只显示既没有标记 visited，也没有标记 captured 的 Portal。
  • Uncaptured Portals： 显示所有尚未标记 captured 的 Portal，其中也包括那些已经访问过但还没有占领的 Portal。
  • 两个图层同时开启： 只要 Portal 满足其中任一条件，就会继续显示。这个模式很适合在保留所有未占领机会的同时，把完全没碰过的目标一起留在图上。

如何使用 Portal Coverage Layers

  1. 先安装并启用 Uniques Tools；如果你本来就在用原版 Uniques，也可以直接使用。
  2. 确保你的 Portal 历史已经在所使用的 uniques 插件中录入，可以是手动记录，也可以来自导入或同步功能。
  3. 安装并启用 Portal Coverage Layers。
  4. 打开 IITC 图层列表，按需启用 Unvisited Portals、Uncaptured Portals 或两者同时启用。
  5. 把地图上仍然可见的 Portal 直接当作补点、清理或跑图目标使用。

为什么它比热力图更直观

热力图适合看整体区域的密度和趋势，但你仍然要自己去解读颜色强弱和分布重叠。Portal Coverage Layers 则更直接，它会把已经完成的 Portal 从视图里移除，让地图本身变成一份目标清单。

对于补最后一批 uniques、回头清理老区域、或者检查一片区域里是否还有从未访问或从未占领的 Portal，这种显示方式通常会更高效。

依赖与限制

这个插件必须依赖 Uniques Tools 或原版 Uniques。它自己不会抓取官方历史，也不会维护独立数据库，而只是读取 local-plugins/uniques-tools 或 plugins/uniques.js 已经保存好的 Portal 历史。

另外，它的结果只基于当前 IITC 客户端已经加载到地图上的 Portal，以及你本地保存的历史状态，因此更适合作为实战规划视图，而不是全局实时真相。

常见问题

  • 没有 Uniques Tools 也能用吗？ 可以，只要你安装并使用了原版 Uniques。这两个插件至少需要有一个提供 visited 和 captured 历史。
  • Unvisited 和 Uncaptured 有什么区别？ Unvisited 会隐藏 visited 和 captured；Uncaptured 只隐藏 captured。
  • 是不是要在两个插件里分别维护数据？ 不需要。你只需要在当前使用的 uniques 插件里维护历史数据，Portal Coverage Layers 只是读取并显示它。

相关插件

  • Uniques Tools 是这个插件首选的依赖，也是 visited / captured 历史的主要来源；同时也支持原版 Uniques 作为 fallback。
  • Portal Afterimage 可补充本地已知 Portal 的可见性增强，与本插件相辅相成。
  • Player Activity Log 则能补上本地行动历史，让补点和复盘更完整。
